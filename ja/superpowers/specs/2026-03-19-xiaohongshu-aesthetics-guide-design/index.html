    Xiaohongshu Aesthetics Guide — Design Spec¶

    Overview¶

    A MkDocs Material documentation site that teaches Xiaohongshu (RED) creators how to make beautiful posts. Pure visual and copywriting aesthetics — no data analysis, no code, no technical content. The site itself should embody the beauty it teaches.

    Target Audience¶

    • Beginner Xiaohongshu creators learning from scratch
    • Experienced creators looking to elevate visual quality and click-through rates

    Tech Stack¶

    • Framework: MkDocs Material (same as rec_system reference project)
    • i18n: mkdocs-static-i18n plugin, default language Chinese, toggle to English
    • File convention: xxx.zh.md / xxx.en.md pairs
    • Package manager: uv
    • Font: Geist Sans only (no Geist Mono — this is not a code-focused site)
    • Deployment: GitHub Pages via mkdocs gh-deploy
    • No: Jupyter, MathJax, ML libraries, notebook plugins

    Visual Style¶

    Based on rec_system's extra.css with the following changes:

    Property rec_system producer_beauty
    Accent color (light) #0070f3 blue #E8A0BF rose pink
    Accent color (dark) #3b82f6 blue #F0B6D0 light rose
    Link color blue rose pink
    Pipeline gradient blue-purple rose pink gradient (#F5D5E0 to #C97191)
    btn-primary black black (rose pink as accent only)

    Everything else stays the same: Geist font, layout, cards, tables, border styles, light/dark toggle, responsive design.

    CSS Custom Properties (full set)¶

    Light mode ([data-md-color-scheme="default"]): - --accent: #E8A0BF - --md-accent-fg-color: #E8A0BF - --md-typeset-a-color: #E8A0BF - All other properties same as rec_system

    Dark mode ([data-md-color-scheme="slate"]): - --accent: #F0B6D0 - --md-accent-fg-color: #F0B6D0 - --md-typeset-a-color: #F0B6D0 - All other properties same as rec_system

    Pipeline stage colors (light): - stage-cover: background: #FCE4EC; color: #880E4F - stage-color: background: #F8BBD0; color: #AD1457 - stage-imagery: background: #F48FB1; color: #C2185B - stage-copy: background: #EC407A; color: #fff - stage-identity: background: #C97191; color: #fff

    Pipeline stage colors (dark): same hues, adjusted for dark background.

    Content Structure¶

    Part 1: Cover Design (封面设计)¶

    • 1.1 Composition (构图法则) — rule of thirds, symmetry, leading lines, white space
    • 1.2 Typography (字体与排版) — font selection, size ratios, hierarchy
    • 1.3 Layout (信息层级与布局) — visual hierarchy, information density

    Part 2: Color (配色)¶

    • 2.1 Color Theory (色彩基础) — color wheel, hue/saturation/brightness
    • 2.2 Style Palettes (风格色板推荐) — Morandi, high-saturation, cream, dark/gold
    • 2.3 Matching Rules (搭配法则) — 60-30-10 rule, contrast, harmony
    • 2.4 Tools & Practice (配色工具与实操) — apps and workflows

    Part 3: Imagery (配图)¶

    • 3.1 Selection (选图原则) — what makes a good image, subject prominence
    • 3.2 Editing (修图与滤镜) — editing principles, filter consistency
    • 3.3 Arrangement (多图编排) — multi-image post sequencing
    • 3.4 Cover Types (不同类型封面图) — person, flatlay, comparison, text poster, scene

    Part 4: Copywriting Aesthetics (文案美学)¶

    • 4.1 Titles (标题的美感) — title formulas, rhythm, visual appeal
    • 4.2 Body Text (正文排版节奏) — paragraph rhythm, emoji usage, information density
    • 4.3 Keywords (关键词的优雅嵌入) — natural keyword placement without stuffiness

    Part 5: Visual Identity (个人视觉风格)¶

    • 5.1 Consistency (建立视觉一致性) — fixed color tone, font, layout habits
    • 5.2 Finding Your Style (找到你的风格) — style references, mood boards
    • 5.3 Evolution (风格的迭代与进化) — how to evolve without losing identity

    Total: 17 tutorials x 2 languages = 34 content files

    Agent Team¶

    Agent Role
    Main (coordinator) Build structure, write content, coordinate
    Designer Review each completed module for visual consistency, color harmony, layout quality, content clarity, and zh/en alignment

    Workflow¶

    1. Main builds project skeleton (mkdocs.yml, extra.css, pyproject.toml, Makefile)
    2. Designer reviews skeleton
    3. Main writes homepage
    4. Designer reviews homepage
    5. For each Part: Main writes content → Designer reviews
    6. Final full-site review by Designer

    Designer Review Criteria¶

    • Visual consistency (colors, spacing, fonts unified)
    • Content quality (tutorials clear, examples beautiful)
    • User experience (navigation clear, reading rhythm comfortable)
    • zh/en content alignment
