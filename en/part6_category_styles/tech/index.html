    Four tech post styles: Vibe Coding, indie dev monetization, AI tool comparison, dev productivity tools. They all share one thing: dark background + bold title + minimal layout.

    Cover Style¶

    Core formula: dark background + bright large title + tech tags (language/framework). Never screenshot raw code as the cover — put the result, not the process, on the cover.

    Color Schemes¶

    Three classic tech palettes — pick one and stick with it:

    GitHub Dark¶

    Background #0D1117
    Accent #58A6FF
    Text #F0F6FC

    The safest choice, perfect for open source projects and GitHub-related content. Blue accent naturally conveys "professional + trustworthy."

    Dracula¶

    Background #282A36
    Accent #BD93F9
    Text #F8F8F2

    Purple feels more creative and personal. Great for indie hackers, side projects, and creative dev content.

    Solarized¶

    Background #002B36
    Accent #859900
    Text #FDF6E3

    Green tones feel more "geeky." Best for low-level tech, framework comparisons, and hardcore tutorials.

    Title Formulas¶

    Viral tech titles always highlight outcomes, not process:

    • "I built Y with X" — I built an auto-report tool with Claude API
    • "Built Y in X days" — Built an AI tool in 3 days, open sourced
    • "This X is insane" — This open source tool is insanely good
    • "X vs Y, which one?" — React vs Vue, which to pick in 2025?

    Post Structure¶

    The golden structure for tech posts — keep every section short:

    1. Why I built it — Tell a story, 1-2 sentences on the pain point
    2. Show the result — Screenshots/GIFs/video, this is the most important part
    3. Tech stack — Brief list of what you used, not a tutorial
    4. Links — GitHub URL, live demo

    Remember

    Xiaohongshu is not Stack Overflow. Users come here to discover cool things, not to learn programming. Making non-technical people understand and think it's cool is the key to tech posts going viral.
